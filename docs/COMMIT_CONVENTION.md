# 커밋 규칙

## 형식

```text
타입: 제목

- 변경 내용 1
- 변경 내용 2
- 변경 내용 3
```

## 예시

```text
feat: 회사 검색 기능 추가

- 검색 입력 컴포넌트 구현
- 자동완성 드롭다운 구현
- 엔터 선택 기능 추가
```

```text
fix: 포스트 모달 스크롤 오류 수정

- 모달 높이 제한 추가
- 내부 스크롤 적용
```

## 타입

- feat : 기능 추가
- fix : 버그 수정
- refactor : 코드 개선
- style : UI 및 스타일 수정
- docs : 문서 수정
- test : 테스트 코드
- chore : 설정 및 패키지 작업

## 규칙

- 하나의 커밋은 하나의 목적만 가진다.
- 제목만 보고 작업 내용을 알 수 있어야 한다.
- 변경 내용이 여러 개라면 목록으로 작성한다.
- 의미 없는 메시지는 사용하지 않는다.
